在 Jetty 嵌入式中使用注解

Jetty 支持 servlet 规范注解.默认不启用, 所以下面的部分向你展示了如何启用它,并如何使用它们.

快速设置

嵌入

Jetty 分发包

如果你正在使用 jetty 分发包, 默认启用了注解. annotations module 和它的传递依赖负责注解处理.

注意与 JNDI 相关的注解, 比如 @Resource 和 @Resources 是通 span class="bold">jndi 模块启用的, 它是 annotations 模块的传递依赖(译者注: annotations -> plus -> jndi), 因此对于分发包默认是启用的.

Jetty Maven 插件

注解和 JNDI 对于 maven 插件是预先启用的.

嵌入

为了在嵌入式环境中使用注解,你需要在你的 classpath 中包含 jetty-annotations jar 和它所有的依赖, 你也需要将 org.eclipse.jetty.annotations.AnnotationConfiguration 包含到 配置类 中来应用到代表你的 webpp 的 org.eclipse.jetty.webapp.WebAppContext.

这里是一个示例应用程序以嵌入式的形式设置了来自分发包中的标准的 test-spec.war webapp. 它可以在 jetty git 仓库的 examples/embedded 项目中找到. 注意 test-spec.war 不只是使用了注解, 也使用了 a class="link" href="jndi.html" title="Chapter 11. Configuring JNDI">JNDI, 所以这个示例也启用了它们的处理(通过 org.eclipse.jetty.plus.webapp.EnvConfiguration, org.eclipse.jetty.plus.webapp.PlusConfiguration 和它们相关的 jars).
