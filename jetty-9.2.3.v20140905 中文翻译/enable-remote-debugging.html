远程调试

如果你有一个部署到 Jetty 中的 web 应用程序,你可以很容易地从一个调试的角度远程地与它交互. 基础要点就是你必须使用额外的参数启动远程 JVM,然后在 Eclipse 中启动一个远程调试会话用于 webapp 的问题.这个很容易完成.

注意

这个例子假设你正在 jetty-分发包中部署你的 web 应用程序.

启动 Jetty

假设你在 jetty 中部署你的 webapp, 有两种不同的方式处理这个问题:

通过命令行

在命令行中添加需要的参数像这样.

$ java -Xdebug -agentlib:jdwp=transport=dt_socket,address=9999,server=y,suspend=n -jar start.jar
通过 start.ini

使用这种方式是最好的,如果你想调试一个特定的 jetty-分发包,并且不需要记住命令行.

  1. 编辑 start.ini 并且取消注释 --exec 行, 这是必须的,如果你想添加 jvm 选项到 start.ini 文件,因为 jetty-start 必须生成必须的 classpath,并 fork 一个新的 jvm.
