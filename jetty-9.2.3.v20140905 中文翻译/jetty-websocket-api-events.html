WebSocket 事件(Events)

每个 WebSocket 可以接收各种事件:

Connect 事件

WebSocket 的一个依据是 Upgrade 已经成功, 此时 WebSocket 打开了.

你将接收一个 org.eclipse.jetty.websocket.api.Session 对,它引用了用于这个打开事件(Open Event) 的特定 session.

对于普通的 WebSockets,持有这个 Session 并使用它用于与远程端点通信很重要.

对于无状态的 WebSockets,Session 将被传递到每个事件当它发生的时候,允许你用1个 WebSocket 的实例服务多个远程端点.

Close 事件

表明 WebSocket 现在是关闭的.

每个 Close 事件将有1个状态码(和一个可选的关闭原因消息)

一个普通的 WebSocket 关闭将经历一个关闭握手,本地端点和远程端点都发送一个关闭帧(close frame)表明连接要被关闭.

有可能本地的 WebSocket 通过发出一个 Close 帧给远程端点来表明它期望关闭, 但是远程端点可用继续发送消息直到它发送了一个 Close 帧.这是总所周知的半关闭(Half-Open)连接, 注意一旦本地端点已经发送了 Close 帧,它不能再写入任何的 WebSocket 信息流.

对于一个异常的关闭,比如连接断开或者连接超时,底层的连接将被终止而不会经历一个 Close 握手,这个将会导致一个关闭事件 (很可能是一个相应的错误事件).

错误事件

如果在执行期间, 一个错误发生, WebSocket 将通过这个事件 handler 被通知

消息事件

表明完整的消息已经被接收到,准备好被你的 WebSocket 来处理.

这个可以是一个 (UTF8) 消息文本或一个原始的二进制消息.
