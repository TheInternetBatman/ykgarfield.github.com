创建一个新的 Jetty Base

上面描述的demo-base 目录是 jetty.base 机制的一个示例,在Jetty 9.1中加入. 一个Jetty base 允许一个服务器的实例配置和 web 应用程序从 jetty 二进制包中分离而被存储, 这样升级的时候具有较小的破坏.Jetty 的默认的配置基于以下两个属性：

jetty.home

这个属性定义了 jetty 二进制包的位置,它的libs, 默认的modules和默认的XML文件(通常为start.jar,lib,等)

jetty.base

这个属性定义了一个指定的jetty服务器实例的位置,它的配置,logs和web应用程序(通常为start.ini,start.d,logs和webapps)

jetty.home 和 jetty.base 属性可以明确地在命令行中设置,或者它们也可以从环境中被推断出来,如果使用下面类似的命令行： 

					
> cd $HOME/my-base
> java -jar $HOME/jetty-distribution-9/start.jar

下面的命令创建一个新的base目录,然后启用一个HTTP连接器和web应用程序发布器:

					
> mkdir /tmp/mybase
> cd /tmp/mybase

更改 Jetty 端口

你可以通过在命令行中设置 jetty.port 属性配置Jetty运行于一个不同的端口： 

					
> cd $HOME/jetty-distribution-9/demo-base
> java -jar start.jar jetty.port=8081

或者,属性值可以从 start.ini 文件和 start.d/*.ini 文件被加入到有效的命令行中.默认情况下,jetty 二进制包在 start.d/http.ini 文件中定义 jetty.port 属性,可以被编辑设置为另一个值.

注意

属性配置经由以下的链生效：

  • start.d/http.ini 文件是有效命令行的一部分,包含了 --module=http 参数,会激活 http 模块

  • modules/http.mod 文件定义了 http 模块, 指定了 etc/jetty-http.xml 配置文件和它使用的模板 ini 属性.

  • jetty.port 属性由etc/jetty-http.xml的Property XML 元素使用去注入给ServerConnector实例

更多的信息查看 快速启动配置入门 和 配置连接器.

启动 HTTPS

为了添加 HTTPS connector 到 jetty 配置中,https 模块可以通过以下命令激活:

					
> java -jar start.jar --add-to-startd=https
https           initialised in ${jetty.home}/start.d/https.ini (created)
ssl             initialised in ${jetty.home}/start.d/ssl.ini (created)
server          enabled in     ${jetty.home}/start.ini
resources       enabled in     ${jetty.home}/start.ini
ext             enabled in     ${jetty.home}/start.ini

> java -jar start.jar
2013-09-06 13:52:43.326:INFO:oejs.Server:main: jetty-9...
[...]

--add-to-startd 命令在 ini 文件中设置了有效的命令行,如下运行一个 https 连接:

  • 创建 https.ini 文件,激活和配置 https连接器模块. https 模块将 etc/jetty-https.xml 加入到有效的命令行中.

  • 创建 start.d/ssl.ini 文件,激活和配置SSL keystore,使用示例密码.ssl 模块将 etc/jetty-ssl.xml 加入到有效的命令行中.

  • 检查 etc/keystore 文件是否存在,如果不存在,下载一个示例 keystore 文件.

注意

如果一个start.ini文件优先于个别的 start.d/*.ini 文件,可以将 --add-to-start=module 追加到模块中去激活 start.ini 文件而不是在 start.d 中创建一个文件.

更改 Jetty HTTPS 端口

你可以在命令行中通过设置 https.port 属性配置 SSL connector 运行于一个不同的端口 

						
> cd $HOME/jetty-distribution-9/demo-base
> java -jar start.jar https.port=8444

另外,属性值可以从 start.ini 文件和 start.d/*.ini 文件构建被添加到有效命令行中.如果你使用 --add-to-startd 命令启用 https,然后你可以在 start.d/https.ini 文件中编辑这个值. 如果你使用 --add-to-start 命令,你可以在 start.ini 文件中编辑这个属性.

更多的 start.jar 选项

start.jar 的工作是解释命令行, start.ini 和 start.d 去构建 Java classpath 和属性列表,将配置文件传递给 Jetty XML 配置机制的 main 类. start.jar 机制有许多选项,文档在 startup 管理章节,可以通过使用以下的命令看下它们的使用总结： 

> java -jar start.jar --help
