context path 是一个 URL 路径的前缀,用来选择 contexts(s),也就是已经通过的传入的请求.通常一个 URL 在一个 Java Servlet 服务器中的格式为 http://hostname.com/contextPath/servletPath/pathInfo, 每个路径元素可以是0个或多个 / 分割的元素.如果没有 context path,context 被称为 根(root) context. root context 必须被配置为 "/", 但是通过 servlet API getContextPath() 方法被报告为空字符串.

如何设置 context path 依赖于你如何部署 web 应用程序(或 ContextHandler):

使用嵌入式 Deployment

如果你从代码运行 Jetty 作为一个嵌入式的服务器(见嵌入式), 设定 context path 就是在 ContextHandler 实例(或 WebAppContext 实例) 上调用 setContextPath 方法.

通过命名约定

如果一个 web 应用程序通过使用 DeploymentManager 的 WebAppProvider 而不是一个 XML IoC 文件部署, 这样 WAR 文件的名称被用来设置 context path:

  • 如果 WAR 文件名为 myapp.war, context 将使用一个 /myapp 的 context path 被部署.

  • 如果 WAR 文件名为 ROOT.WAR(或任何大小写不敏感的变体), context 将使用一个 / 的 context path 被部署.

  • 如果 WAR 文件名为 ROOT-foobar.war(或任何大小写不敏感的变体), context 将使用一个 / 的 context path 和 "foobar" 的虚拟主机被部署.
