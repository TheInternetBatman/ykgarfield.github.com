Jetty POJO 配置

Jetty的核心组件简称为简单的Java对象(POJOs); 配置Jetty的处理大部分为实例化,装配和在Jetty POJO上设置字段的处理,你可以通过以下方式做到这点：

  • 编写Java代码直接实例化和装配Jetty对象.这个被称为嵌入式的Jetty.

  • 使用Jetty XML配置,这是一种控制反转(IoC)框架.作为XML对象实例化和装配Jetty对象. etc/jetty.xml 文件是主要的Jetty XML配置文件,但是还有很多etc/jetty- feature .xml 文件包含在jetty分发包中.

  • 使用第三方的IoC框架,比如Spring, 作为Spring beans实例化和装配Jetty对象.

因为主要的Jetty配置是通过IoC完成的,Jetty API文档是最终的配置参考.

Jetty 启动配置文件

Jetty分发包通过 start.jar 机制使用下面的配置文件去实例化,注入和启动服务器.

ini 文件

Jetty启动机制使用命令行, start.ini 文件和所有的 start.d/*.ini 文件去创建一个有效的命令行参数.参数可以是：

  • Jetty IoC(或Spring)XML格式的XML文件

  • 以 --module=name 的形式激活模块

  • name=value 形式的属性,用于参数化Jetty IoC XML

  • 标准的Java属性文件包含了额外的启动属性.

  • 其它的 start.jar 选项(见 java -jar start.jar -- help)

  • 一些JVM选项

Ini 文件位于 jetty.base(如果不同于 jetty.home),通常会被编辑去修改配置.

mod files

modules/*.mod 文件包含了指定的模块,可通过 --module=name 激活.每个mod文件定义了：

  • 模块依赖用于排序和激活

  • 需要被模块添加到 classpath 的库

  • 需要被模块添加到有效命令行的 XML 文件

  • 激活的模块所需要的文件

  • 使用模板 ini 文件当使用 --add-to-start=name 选项的时候

Mod 文件通常位于 jetty.home, 但是如果需要的话可以在 jetty.base 中覆盖.通常 module 文件很少被编辑,只用于重要结构上的修改.

XML 文件

在命令行, ini 文件或通过 module 定义被添加到有效命令行中的 Jetty 或 Spring IoC 格式的 XML 文件. XML 文件实例化和注入实际的 Java 对象构成了 server, connectors 和 contexts. 因为 Jetty IoC XML 文件频繁使用属性,许多通用的配置任务可以不用编辑这些 XML 文件完成. 如果 XML 配置修改是必要的, XML 文件应该从 jetty.home/etc 拷贝到 jetty.base/etc 在被修改之前.

其它的配置文件

除了上面描述的启动配置文件外.服务器的配置可以使用下面的文件:

Context XML 文件

任何在 webapps 目录中找到的 Jetty 或 spring IoC 格式的 XML 文件可以被 deploy 模块使用去实例化和注入 HttpContext 实例去创建一个指定的 context. 这些可以是标准的 web 应用或从特定目的的 handlers 定制的 contexts.

web.xml

Servlet 规范定义了 web.xml 部署描述符,定义和配置 过滤器,servlets,和一个 web 应用 使用的资源. Jetty WebAppContext 组件使用这种 XML 格式去:

  • 设置一个 web 应用 context 的默认的配置 .

  • 解释在 WEB-INF/web.xml 文件中一个 web 应用提供的特定的配置.

  • 解释在包含在 WEB-INF/lib 中的 Jar 文件的 META-INF 目录中的描述符 fragments.

通常用于一个 web 应用的 web.xml 文件可以在 WAR 文件/目录的 WEB-INF/web.xml 位置或在 WEB-INF/lib 中找到的作为 jar 文件的 web.xml fragments. Jetty 也支持多个 web.xml 文件, 这样一个默认的描述符可以被应用在 WEB-INF/web.xml(通常通过 deploy 模块设置到 etc/webdefault.xml)之前 和一个覆盖的描述符可以在 WEB-INF/web.xml 之后被应用(通常通过一个 context XML 文件设置, 见 test.xml).

Property 文件

标准 Java 属性文件 也可通过以下几种方式用于 Jetty 配置:

  • 通过 Property 元素的使用参数化 Jetty IoC XML.

  • 为了配置默认的日志机制(StdErrLog).你可以插入其它的日志框架,并使用属性文件(比如,log4j).

  • 作为一个简单的数据库用于登录名和认证.

配置 Jetty 分发包

使用一个 Jetty 正常的分发包, 上面介绍的配置机制通常使用如下:

$JETTY_HOME/start.ini

用于全局的命令行选项.在标准的 Jetty 分发包中,从版本 9.1 开始, start.ini 的内容已经被移动到了 start.d/ 下的模块化 ini 文件中.

$JETTY_HOME/start.d

模块化的 ini 的目录用于设置主要的 Jetty 模块的选项,参数和配置文件.这些模块可以通过重命名而启用/禁用或 使用 start.jar 的 --enable 或 --disable 选项.

$JETTY_HOME/etc/*.xml

Jetty IoC XML 文件配置独立的特性;比如 jetty.xml (用于服务器), jetty-http.xml, jetty-https.xml, jetty-jmx.xml .

$JETTY_HOME/webapps/*

标准 WAR 文件, web应用和 context IoC XML 文件被部署的目录.
