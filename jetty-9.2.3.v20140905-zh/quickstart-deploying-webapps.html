部署 Web 应用程序

Jetty 演示 Web 应用程序

配置了 deploy 模块的 Jetty 服务器实例将有一个 web 应用程序部署器, 热部署在 webapps 目录中找到的文件. 你放置在 webapps 目录中的标准的 WAR 文件和 jetty 配置文件将使用下面的约定被热部署到服务器中:

  • 一个名为 example/ 的目录作为标准的 web 应用程序被部署, 如果它包含一个 WEB-INF/ 子目录, 否则它被当做静态内容的 context 被部署.context 路径是 /example (也就是 http://localhost:8080/example/), 除非基名称是 ROOT(忽略大小写),这种情况下 context 路径是 /.如果目录名称以 ".d" 结束,它将被忽略(但是可通过明确的配置被使用)

  • 一个名为 example.war 的文件作为标准的 web 应用程序被部署, context 路径是 /example(也就是 http://localhost:8080/example/). 如果基目录为 ROOT(忽略大小写), context 路径是 /.如果 example.war 和 example/ 都存在, 只有 WAR 被部署(可能会使用这个目录作为一个解压的位置).

  • 一个 XML 文件比如 example.xml 被部署作为一个 context(配置通过 XML 定义). 配置自身必须设置 context 路径.如果 example.xml 和 example.war 都存在. 只有 XML 被部署(但是可能在它的配置中使用 WAR).

如果你有一个 标准的 web 应用,你可以将它热部署到 Jetty 中通过拷贝它到 webapps 目录.

Jetty 示例 Web 应用程序

demo-base/webapps 目录包含了以下的可部署和辅助的文件:

ROOT/

一个静态内容的目录被部署到根 context /, 因为它的名称.包含了 Jetty 示例欢迎页.

test.xml

一个 context 配置文件配置和部署 test.war. 额外的配置包含了 context 路径同时设置额外的在 test.d 目录中找到的描述符.

test.war

演示 web 应用, 通过 test.xml 配置和部署.

test.d

一个包含了被 test.xml 使用的额外的配置文件, 注入额外的配置到 test.war.

async-rest.war

一个 异步 REST 到 eBay 的 web 应用示例, 基于文件名称自动部署到 /async-rest.

test-jaas.war

一个演示 web 应用利用 JAAS 用于认证.

test-jaas.xml

一个 context 配置文件配置 test-jaas.war. 额外的配置包含了设置 LoginService 用于认证(authentication)和授权(authorization).

test-jndi.war

一个演示 web 应用展示了 JNDI 的使用.

test-jndi.xml

一个 context 配置文件配置 test-jndi.war. 额外的配置包含了定义了可以从 webapp 引用的在名称空间的对象.

test-spec.war

一个演示 web 应用展示了 annotations, fragments, ServletContainerInitializers, 和其它的 Servlet 规范 3.0/3.1 特性.

test-spec.xml

一个演示 web 应用展示了 annotations, fragments, ServletContainerInitializers, 和其它的 Servlet 规范 3.0/3.1 特性.

xref-proxy.war

一个演示 web 应用,使用了透明代理(transparent proxy) 服务来自 Eclipse 的 站点 的 Jetty src xref.

example-moved.xml

一个演示 context 配置文件, 展示了如何使用 MovedContextHandler 从一个路径重定向到另一个.
