使用

Jetty 目前有两种层次的请求统计收集(request statistic collection):

  • AbstractConnector 类的子类可以可选择地收集关于连接的统计,及请求的数目.

  • StatisticsHandler 类可以用来收集请求统计.

除了这个, AbstractSessionHandler 类的子类也可以选择性地收集 session 统计.

AbstractConnector 和 StatisticsHandler(文档中为 AbstractSessionHandler, 应该是写错了) 默认是关闭的, 必须通过手工为每个实例配置或者通过 JMX 接口打开.Statistics Handler 不包含在默认的 Jetty 配置中,需要手工配置.

观察统计

为了观察统计, 你需要能够连接到 Jetty 使用 JConsole 或一些其它的 JXM 代理.查看 使用 Java 管理扩展 (JMX) 获取更多细节.

Connector 统计

关于连接持续时间(connection) 和请求数目的详细统计只有当一个连接被关闭的时候才会被整理. 连接的当前和最大数目只是现场统计(live statistics).为了学习关于如何打开 connector 统计,请参考 Jetty 统计教程, 尽管这个不被推荐,且最好使用 JMX 代理来选择统计只有当需要的时候.

请求统计

为了收集请求统计, StatisticsHandler 必须被配置作为 server 的 handlers 中的其中一个. 通常这个可以通过顶层的 handler 完成,但是你可以选择只为一个 context 配置一个 statistics handler, 通过创建一个 context 配置文件. 请注意 jetty-stats.xml 在命令行中必须出现在主 Jetty 配置文件之后,如下所示. 它可以在 start.ini 文件中被注释掉. 


$ java -jar start.jar OPTIONS=default etc/jetty.xml etc/jetty-stats.xml

Session Statistics

Session 处理内置在 Jetty 用于任意的 servlet 或 webapp context. 关于 session 持续时间的详细统计只有当一个 session 被关闭的时候才会整理. session 的当前,最小和最大数目只是现场统计("live" statistics).session 统计默认是启用的, 不需要配置.
