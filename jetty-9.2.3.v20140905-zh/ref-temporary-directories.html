Jetty 自身没有临时目录,但是你可以为每一个 web 应用程序分配一个目录,WAR 在此解压,运行过程中被编译的 JSPs, 等.如果你没有指定一个特定的临时目录, Jetty 将按需创建一个当你的 web 应用程序启动的时候.无论你设置了临时目录的位置-或你让 Jetty 为你创建-你也可以有机会保持或删除临时目录当 web 应用程序停止的时候.

默认的临时目录

默认情况下,Jetty 将为每一个 web 应用程序创建一个临时目录.目录的名称是以下是格式: 

"jetty-"+host+"-"+port+"-"+resourceBase+"-_"+context+"-"+virtualhost+"-"+randomdigits+".dir"

比如:

jetty-0.0.0.0-8080-test.war-_test-any-8900275691885214790.dir

0.0.0.0 为主机地址, 8080 是端口, test.war 是 resourceBase, test 是 context path (/ 转换为 to _), any 是虚拟主机, randomdigits 是保证唯一的数字字符串.

一旦临时目录 被创建,它将作为 context 属性 javax.servlet.context.tempdir 的值,可以被取回.

"work" 目录

主要是为了向后兼容,从 jetty-9.1.1 开始,可以在 ${jetty.base} 目录创建一个 "work" 目录.如果这样的一个目录被找到, 它将被假定为你想使用它作为在那个 ${jetty.base} 中所有 webapps 的临时目录.此外,作为历来的情况,这些位于 work 目录的临时目录不会被清理 当 jetty 存在的时候(更准确的来说,一个 context 对应的临时目录不会被清理当 context 停止的时候).

当一个 work 目录被使用,特定 context 临时目录名称生成的算法将忽略随机数字字符串.这个保证了目录的名称在 context 重启的时候保持一致.

持久化临时目录

有时候你可能发现在 web 应用程序重启之间保持临时目录的内容是非常有用的.默认情况下,Jetty 将 不会 持久化临时目录.为了让 Jetty 可以保持它,使用 WebAppContext.setPersistTempDirectory(true).

注意如果你调用 setPersistTempDirectory(true),但是让 Jetty 每次创建一个新的临时目录(也就是,你没有设置一个明确的临时目录), 这样将在你选择的临时目录位置累积临时目录.
