管理 Jetty Base 和 Jetty Home

Jetty 分发包中的 Demo-Base
声明 Jetty Base

从 Jetty 9.1 开始,可用维护独立的 Jetty 二进制安装(也称作 ${jetty.home}), 和为你特定的环境自定义(也称作 ${jetty.base}) 的分离.

Jetty Base

也被称为 ${jetty.base} 属性

这是你配置和自定义 Jetty 分发包的位置.

Jetty Home

也被称为 ${jetty.home} 属性

这是 Jetty 分发包二进制,默认 XML IoC 配置,和默认模块定义的位置.

潜在的配置从这2个目录位置解析.

配置解析规则

检查 Jetty Base

如果相对于定义的 Jetty Base 的引用配置存在,那么就使用它.

检查 Jetty Home

如果相对于定义的 Jetty Home 的引用配置存在,那么就使用它.

使用 java.io.File(String pathname) 逻辑

最后,使用引用作为一个 java.io.File(String pathname) 引用, 下面是哪个构造函数的默认解析规则.

简单地说,引用将被用作:相对(当前工作目录, 又叫做, ${user.dir} 或绝对路径,或者甚至是网络应用(比如 在 Windows,并使用 UNC 路径)).

对于关于使用 start.jar 是如何工作的更多细节,查看 使用 start.jar: 执行

如果你看下 ${jetty.base}/start.ini,你就爱那个看到某些如下的内容. 

[my-base]$ cat start.ini
# Enable security via jaas, and configure it
--module=jaas
jaas.login.conf=etc/login.conf

# Enable rewrite examples
--module=rewrite
etc/demo-rewrite-rules.xml

# Websocket chat examples needs websocket enabled
# Don't start for all contexts (set to true in test.xml context)
org.eclipse.jetty.websocket.jsr356=false
--module=websocket

# Create and configure the test realm
etc/test-realm.xml
demo.realm=etc/realm.properties

# Initialize module server
--module=server
threads.min=10
threads.max=200
threads.timeout=60000
jetty.dump.start=false
jetty.dump.stop=false

--module=deploy
--module=jsp
--module=ext
--module=resources
--module=client
--module=annotations

${jetty.base}/start.ini 是 Jetty 的主要启动配置的入口点.在这个例子中,你将看到我们正在为 Jetty 启用一些模块,指定某些属性, 也引用了某些 Jetty IoC XML 文件(也就是 etc/demo-rewrite-rules.xml 和 etc/test-realm.xml 文件).

当 Jetty 的 start.jar 解析 start.ini 中的条目,它将遵循 上面的解析规则.

比如,etc/demo-rewrite-rules.xml 的引用被发现在 ${jetty.base}/etc/demo-rewrite-rules.xml.

声明 Jetty Base

Jetty 分发包的 start.jar 是管理这种分离星期的组件.

Jetty start.jar 和 XML 文件总是假定 ${jetty.home} 和 ${jetty.base} 是定义的,当启动 Jetty 的时候.

你可以选择手动定义 ${jetty.home} 和 ${jetty.base} 目录,比如这样: 

[jetty-distribution-9.2.3.v20140905]$ pwd
/home/user/jetty-distribution-9.2.3.v20140905
[jetty-distribution-9.2.3.v20140905]$ java -jar start.jar \
    jetty.home=/home/user/jetty-distribution-9.2.3.v20140905 \
    jetty.base=/home/user/my-base
2013-10-16 09:08:47.802:INFO:oejs.Server:main: jetty-9.2.3.v20140905
2013-10-16 09:08:47.817:INFO:oejdp.ScanningAppProvider:main: Deployment monitor [file:/home/user/my-base/webapps/] at interval 1
...

或你可以声明一个目录,然后让另一个被发现.

下面的示例使用默认发现的 ${jetty.home},也就是使用通过 start.jar 自身所在的父目录作为 ${jetty.home}, 并手工声明了 ${jetty.base}. 

[jetty-distribution-9.2.3.v20140905]$ pwd
/home/user/jetty-distribution-9.2.3.v20140905
[jetty-distribution-9.2.3.v20140905]$ java -jar start.jar jetty.base=/home/user/my-base
2013-10-16 09:08:47.802:INFO:oejs.Server:main: jetty-9.2.3.v20140905
2013-10-16 09:08:47.817:INFO:oejdp.ScanningAppProvider:main: Deployment monitor [file:/home/user/my-base/webapps/] at interval 1
...

但是 Jetty 推荐你总是在你的 ${jetty.base} 目录中启动 Jetty,然后通过远程引用 start.jar 启动 Jetty.

下面演示了这个,通过允许定位 start.jar 而使用默认发现到的 ${jetty.home}, 并使用 user.dir 系统属性作为 ${jetty.base}. 

[jetty-distribution-9.2.3.v20140905]$ pwd
/home/user/jetty-distribution-9.2.3.v20140905
[jetty-distribution-9.2.3.v20140905]$ cd /home/user/my-base
[my-base]$ java -jar /home/user/jetty-distribution-9.2.3.v20140905/start.jar
2013-10-16 09:08:47.802:INFO:oejs.Server:main: jetty-9.2.3.v20140905
2013-10-16 09:08:47.817:INFO:oejdp.ScanningAppProvider:main: Deployment monitor [file:/home/user/my-base/webapps/] at interval 1
...

重要

注意 user.dir 系统属性,它只能在JVM启动时被安全设置,许多第三方的库(特别是日志)使用这个系统属性.

强烈推荐你位于这个目录,也就是你期望的 ${jetty.base} 当启动 Jetty 的时候和 user.dir 的系统属性有一致的行为. (It is strongly recommended that you sit in the directory that is your desired ${jetty.base} when starting Jetty to have consistent behavior and use of the user.dir system property.)
