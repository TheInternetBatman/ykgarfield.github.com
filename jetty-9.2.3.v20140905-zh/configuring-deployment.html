Web应用程序剖析

标准的 Jetty 分发包具有部署标准 Servlet 规范 web 应用程序, Jetty 内部 ContextHandler 部署描述符,甚至是两者混合的能力.

Web 应用程序可在部署绑定到 Jetty 上的一个特定 context path 的一组动态(servlets, filters, jsps, 等...)和静态内容, 支持库, 描述型元数据

最终格式和布局由 Servlet 规范定义,你应该查阅官方的 Servlet 规范文档获取理解 Web 应用布局和结果的更多信息, 然而这里将会概要介绍下关于 Jetty 如何看待这些需求.

Web 应用程序可以被打包到一个单独的 Web 归档(WAR 文件)或一个目录树.

/WEB-INF/

Servlet API 定义了此目录用来存储和 Web 应用程序相关的,但不属于 Web 应用程序公共访问部分的任意东西.

如果你有通过你的 Web 应用程序内部访问的内容,但是应该永远不能通过浏览器直接访问, 那么你应该将它们放到这个目录.

/WEB-INF/web.xml

必须的(Required) 部署描述符定义了你的 web 应用程序的各种行为.

/WEB-INF/classes/

用于 Web 应用程序特定编译的 java classes 的位置

/WEB-INF/lib/

JAR 文件(库) 的目录

Jetty 内部的 WebAppClassloader 将首先从 /WEB-INF/classes/, 然后从 /WEB-INF/lib/ 中找到的 jars 文件加载类.
